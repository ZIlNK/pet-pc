# 桌宠设置中心设计规格

## 概述

将桌面宠物应用右键菜单中分散的功能整合到一个统一的设置中心 GUI 中，简化右键菜单和系统托盘菜单，提供更美观、更易用的配置体验。

## 目标

1. 简化右键菜单，只保留"打开设置中心"、"最小化到托盘"、"退出"三个选项
2. 简化系统托盘菜单，只保留"显示桌宠"、"打开设置中心"、"退出"三个选项
3. 提供统一的设置中心窗口，包含：
   - 桌宠管理（列表、新建、配置）
   - 全局设置（运动控制、休息提醒、系统设置、API设置）
4. 支持新建桌宠（用户提供待机形象，自动生成文件夹和初始配置）
5. 支持导入外部桌宠资源包（zip格式）
6. 为现有桌宠添加向左行走、向右行走、缓降形象的配置入口


**新建入口设计：** 采用网格中的特殊卡片形式，而非顶部按钮：

- "📦 导入" 卡片用于导入资源包

- **预览区域**：显示待机动画（GIF/WebP）或静态图片

- **作者信息**：较小字号，灰色
- **状态标签**：
  - 当前使用的桌宠显示"当前使用"（蓝色高亮）+ [配置] 按钮
  - 其他桌宠显示 [切换使用] 按钮

### 卡片样式

- 圆角矩形（border-radius: 12px）
- 浅色背景 + 细边框
- 悬停：轻微阴影，边框加深
- 尺寸：约 180x220px

### 特殊卡片

- **新建卡片**：显示 "+" 图标和"新建"文字，点击打开 NewPetDialog
- **导入卡片**：显示 "📦 导入" 文字，点击打开导入资源包流程

## 新建桌宠流程 (NewPetDialog)

### 对话框内容

```
┌─────────────────────────────────┐
│         新建桌宠            [×] │
├─────────────────────────────────┤
│                                 │
│   桌宠名称*  [____________]     │
│   作者       [____________]     │
│                                 │
│   待机形象*                     │
│   [选择文件...]  [选择图片]    │
│                                 │
│   预览                          │
│   ┌─────────┐                  │
│   │ 选中的 │                  │
│   │ 图片   │                  │
│   └─────────┘                  │
│                                 │
│           [创建]  [取消]        │
└─────────────────────────────────┘
```

### 创建流程

1. 用户填写桌宠名称（必填）
2. 用户填写作者（可选，默认为"用户"）
3. 用户选择待机形象文件（必填，支持 PNG/GIF/WebP/APNG）
4. 点击"创建"：
   - 在 `pets/` 目录创建新文件夹（以桌宠名称命名）
   - 创建 `animations/` 子目录
   - 复制待机形象文件到 `animations/`
   - 生成 `meta.json`：
     ```json
     {
       "name": "用户输入的名称",
       "author": "用户输入的作者",
       "version": "1.0.0",
       "description": "",
       "preview": "idle.png",
       "regular_image": "待机形象文件名",
       "flying_image": "待机形象文件名",
       "rest_animation": "待机形象文件名"
     }
     ```
   - 生成 `config/actions.json`：
     ```json
     {
       "actions": [
         {
           "name": "idle",
           "type": "animation",
           "weight": 1,
           "animation_files": ["待机形象文件名"],
           "enabled": true,
           "config": {}
         }
       ]
     }
     ```
5. 创建成功后刷新列表

### 导入资源包流程

点击"导入"按钮：
1. 打开文件选择对话框，选择 `.zip` 文件
2. 解压到临时目录
3. 验证结构（必须包含 `meta.json` 和 `animations/`）
4. 验证通过后复制到 `pets/` 目录
5. 刷新列表

## 桌宠配置页面 (PetConfigPage)

### 页面布局

页面分为四个区域：

#### 1. 基础形象配置

```
│   基础形象                       [预览当前形象] │
│   ────────────────────────────────────────   │
│   待机形象      [idle.webp    ] [更换]       │
│   缓降形象      [idle.webp    ] [更换]       │
│   向左行走      [未设置       ] [设置]       │
│   向右行走      [未设置       ] [设置]       │
│   休息动画      [hui.webp     ] [更换]       │
```

**meta.json 字段对应：**
- 待机形象 → `regular_image`
- 缓降形象 → `flying_image`
- 向左行走 → `actions.json` walk 动作的 `animation_files[0]`
- 向右行走 → `actions.json` walk 动作的 `animation_files[1]`
- 休息动画 → `rest_animation`

#### 2. 点击区域配置

```
│   点击区域配置                    [+ 添加区域] │
│   ────────────────────────────────────────   │
│   ☑ 启用点击检测                            │
│   ┌───────────────────────────────────────┐ │
│   │ 区域名称  位置    大小    动作   操作 │ │
│   │ 头部     (0.3,0.1) (0.4,0.3) 摸头 [编辑] │
│   │ 身体     (0.2,0.4) (0.6,0.5) 拥抱 [删除] │
│   └───────────────────────────────────────┘ │
```

- 使用相对坐标（0-1）
- 复用现有 `ClickZoneConfigDialog` 逻辑

#### 3. 动作列表

```
│   动作列表                        [+ 添加动作] │
│   ────────────────────────────────────────   │
│   ┌───────────────────────────────────────┐ │
│   │ 名称  类型    动画文件  权重  启用 操作│ │
│   │ idle  animation idle    1     ☑  [编辑] │
│   │ walk  movement  -       1     ☑  [删除] │
│   │ sit   animation sit     1     ☑  [编辑] │
│   └───────────────────────────────────────┘ │
```

- 复用现有 `ActionManagerGUI` 表格形式
- 点击编辑打开类似 `ActionEditDialog` 的对话框

#### 4. 底部操作栏

```
│                        [保存配置]  [取消]    │
```

#### 1. 运动控制

```
│   休息提醒                                  │

```

```
│   ┌───────────────────────────────────────┐│
│   │ 127.0.0.1                      [删除] ││
│   │ ::1                            [删除] ││
│   │                                [+添加] ││
│   └───────────────────────────────────────┘│
```

**配置对应：** `config.json` 的 `api` 部分

- 点击"保存配置" → 写入 `config/user_config.json`

## 视觉风格

### 整体风格：现代简约

- 浅色背景（白色或浅灰）
- 卡片阴影效果
- 圆角按钮（border-radius: 8px）
- 细边框分隔线
- 清晰的视觉层次

### 颜色方案

- 主色调：蓝色（#0078D4）用于选中状态、按钮高亮
- 背景色：白色（#FFFFFF）或浅灰（#F3F3F3）
- 文字色：深灰（#333333）
- 辅助文字：灰色（#888888）
- 边框色：浅灰（#E0E0E0）

### 字体

- 标题：微软雅黑，14px，加粗
- 正文：微软雅黑，12px
- 辅助文字：微软雅黑，10px

## 依赖关系

- `SettingsCenter` 依赖 `ConfigManager`、`PetLoader`、`DesktopPet`
- 各页面模块依赖 `SettingsCenter` 提供的数据传递
- 复用现有组件：`AnimationSelectDialog`、`ActionEditDialog` 的逻辑

## 实现优先级

1. **高优先级**：设置中心主窗口框架 + 左侧导航
2. **高优先级**：桌宠列表页面 + 新建桌宠对话框
3. **中优先级**：桌宠配置页面（基础形象 + 动作列表）
4. **中优先级**：全局设置页面
5. **低优先级**：点击区域配置（可复用现有对话框）
6. **低优先级**：导入资源包功能
7. **低优先级**：右键菜单和托盘菜单简化
